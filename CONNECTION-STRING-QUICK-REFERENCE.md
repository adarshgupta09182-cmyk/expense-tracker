## What It Looks Like

### Example Connection String
```
postgresql://postgres:abc123xyz@containers-us-west-45.railway.app:6543/railway
```

### Breaking It Down
```
postgresql://  ‚Üê Protocol (always this)
postgres       ‚Üê Username (usually "postgres")
:              ‚Üê Separator
abc123xyz      ‚Üê Password (random, generated by Railway)
@              ‚Üê Separator
containers-us-west-45.railway.app  ‚Üê Hostname (Railway's server)
:              ‚Üê Separator
6543           ‚Üê Port (usually 6543 or 5432)
/              ‚Üê Separator
railway        ‚Üê Database name (usually "railway")
```

- Click on the "PostgreSQL" service box

- At the top, you'll see tabs: "Overview", "Connect", "Logs", "Settings"

- You'll see a section with:

- The Database URL is the one you need to copy

## What to Do With the Connection String

### Add to Railway Web Service

1. Click on "web" service (your Node.js app)
2. Click "Variables" tab
3. Click "Add Variable"
4. **Name:** `DATABASE_URL`
5. **Value:** (paste the connection string here)
6. Click "Save" or "Deploy"

### Example
```
Name:  DATABASE_URL
Value: postgresql://postgres:abc123xyz@containers-us-west-45.railway.app:6543/railway
```

## Common Issues

### "I can't find the Connect tab"
- Make sure you clicked on the **PostgreSQL** service, not the web service
- The web service won't have a "Connect" tab

### "The connection string looks different"
- That's normal! Railway generates unique connection strings
- Just copy whatever you see in the "Database URL" field
- Don't edit it or change anything

### "I see multiple connection strings"
- Use the "Database URL" one (not PSQL Command)
- They're the same thing, just formatted differently

### "The password has special characters"
- That's normal! Railway generates secure passwords
- Copy the entire string exactly as shown
- Don't try to edit or escape special characters

## Need Help?

If the connection string doesn't work:

1. **Check Railway Logs**
   - Click on web service
   - Click "Logs" tab
   - Look for any error messages

2. **Verify Connection String**
   - Go back to PostgreSQL service
   - Click "Connect" tab
   - Copy the Database URL again
   - Make sure there are no extra spaces

3. **Check PostgreSQL Service**
   - Make sure PostgreSQL service is running (green status)
   - If it's red/yellow, click it and check the logs

4. **Test Connection**
   - You can test the connection string in terminal:
   ```bash
   psql postgresql://postgres:abc123xyz@containers-us-west-45.railway.app:6543/railway
   ```
   - If it connects, the string is correct
